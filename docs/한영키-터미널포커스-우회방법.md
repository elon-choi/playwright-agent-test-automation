# 한/영 키 입력 시 Cursor에서 터미널로 포커스 이동 - 우회 방법

키바인딩으로는 해결되지 않는 경우, **한/영 키가 Cursor에 전달되는 키 조합을 바꿔서** 터미널 포커스가 실행되지 않게 할 수 있습니다.

## 방법: Karabiner-Elements로 한/영 키를 F18로 보내기

많은 한국어 Mac 키보드에서 한/영 키는 **Ctrl+Space**로 전달됩니다. Cursor가 이 조합을 받으면 터미널로 포커스를 옮기는 동작이 붙어 있을 수 있고, 이 동작은 키바인딩 설정으로는 끄기 어렵습니다.

**한/영 키를 F18로 보내고**, **macOS 입력 소스 전환 단축키를 F18**로 두면:

- macOS는 F18을 받아서 입력 소스만 전환하고
- Cursor에는 F18만 전달되므로 터미널 포커스가 실행되지 않습니다.

### 1. Karabiner-Elements 설치

- https://karabiner-elements.pqrs.org 에서 다운로드 후 설치
- 설치 후 **시스템 설정 > 개인 정보 보호 및 보안 > 입력 모니터링**에서  
  `karabiner_grabber`, `karabiner_observer` 허용

### 2. 한/영 키를 F18로 매핑

1. Karabiner-Elements 실행
2. **Simple Modifications** 탭 선택
3. **Add item** 클릭
4. **From key**:  
   - 한/영 키가 **Caps Lock**으로 잡히면: `Caps Lock`  
   - 또는 **Right Option**, **lang1** 등 실제로 표시되는 키 선택
5. **To key**: `f18` 선택

(한/영 키가 물리적으로 Caps Lock 위치에 있다면 From key는 Caps Lock으로 두면 됩니다.)

### 3. macOS 입력 소스 단축키를 F18로 변경

1. **시스템 설정 > 키보드 > 키보드 단축키 > 입력 소스**
2. **이전 입력 소스 선택** 더블클릭
3. 새 단축키로 **F18** 입력 (키보드에서 F18 키 누르기)
4. **완료**로 저장

이렇게 하면:

- 한/영 키를 누르면 Karabiner가 F18로 변환하고, macOS가 F18로 입력 소스 전환
- Cursor에는 F18만 들어가서 터미널 포커스가 실행되지 않음

### 4. (선택) Cursor에서 F18 무시

혹시 F18에 다른 동작이 붙어 있으면, `keybindings.json`에 다음을 추가해 비활성화할 수 있습니다.

```json
{
  "key": "f18",
  "command": ""
}
```

---

이 우회 방법을 적용한 뒤에도 한/영 키를 누를 때 터미널로 포커스가 이동하면,  
Cursor/앱 쪽 버그로 보는 것이 맞고, Cursor 공식 채널에 버그 리포트를 남기는 것을 권장합니다.
