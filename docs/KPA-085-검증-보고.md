# KPA-085 직접 실행·검증 방법 및 전수 조사 보고

## 1. 직접 실행·검증 가능한 방법

- **Playwright E2E 실행**: Cursor/에이전트 환경에서 `npm run test:085` 실행 시 프로세스 spawn이 `Aborted` 되어, 이 환경에서는 브라우저 테스트를 끝까지 돌릴 수 없음.
- **대안(구현 완료)**: spawn 없이 **Node만 사용하는 검증 스크립트**를 추가함.  
  - 스크립트: `scripts/verify-kpa-085-steps.mjs`  
  - 실행: `npm run verify:085` 또는 `node scripts/verify-kpa-085-steps.mjs`  
  - 동작: steps 폴더 전체 스텝 문구 **중복 전수 조사** + KPA-085 feature에 쓰인 스텝이 **여러 파일에서 매칭되는지** 검사. (브라우저/Playwright 미사용, 파일 읽기·정규식만 사용.)

## 2. 스텝 문구 중복 전수 조사 결과

`npm run verify:085` 실행 시 출력 기준.

| 구분 | 결과 |
|------|------|
| **전체 steps/*.ts 중복** | 3건 발견 |
| **KPA-085 전용 스텝** | "대여권 n장 소장권 n장 > 영역에서 > 아이콘을 클릭한다" 는 **kpa-085.steps.ts에만** 정의됨 (중복 없음) |

### 2.1 전체 중복 3건 (KPA-085와 무관한 기존 중복)

1. **사용자가 우측 상단의 프로필 아이콘을 클릭한다**  
   - 정의 위치: `common.auth.steps.ts`, `kpa-008-zerostep.steps.ts`  
   - KPA-085 feature에서도 이 스텝 사용 → 실행 시 Multiple definitions 발생 가능.
2. **로그인 페이지의 URL이 {string}인지 확인한다**  
   - 정의 위치: `kpa-008-zerostep.steps.ts`, `kpa-008.steps.ts`
3. **정주행 아이콘이 활성화되고, 뷰어 엔드 영역에 노출되던 메뉴들(배너, 베스트 댓글 등)이 노출되지 않는다**  
   - 정의 위치: `kpa-112.steps.ts`, `kpa-127.steps.ts`

### 2.2 KPA-085 feature 스텝 매칭 검증

- feature 스텝 수: 24 (두 시나리오 합계).
- **"사이트에 접속한다"**: step 정의는 `Given("사용자가 {string} 사이트에 접속한다", ...)` 형태. 검증 스크립트는 feature의 문자열을 그대로 비교하므로 `{string}` 패턴과는 매칭되지 않아 "매칭 없음"으로 나올 수 있음 (스크립트 한계).
- **"프로필 아이콘을 클릭한다"**: 위 2.1과 동일하게 2개 파일에서 정의되어 있어, **Multiple definitions 위험** 있음.

## 3. Node 검증 스크립트 추가 요약

| 항목 | 내용 |
|------|------|
| **파일** | `scripts/verify-kpa-085-steps.mjs` |
| **실행** | `npm run verify:085` |
| **입력** | `steps/*.ts` 전체, `features/pcw/06-콘텐츠홈/kpa-085.feature` |
| **출력** | (1) steps 전역 스텝 문구 중복 목록, (2) KPA-085 feature 스텝별 매칭 여부·다중 정의 여부 |
| **종료 코드** | 중복 또는 매칭 이상 시 1, 정상 시 0 |

스크립트는 **child_process spawn 없이** Node만으로 동작하므로, Cursor/에이전트 환경에서도 실행 가능함.

## 4. 정리

- **직접 실행·검증**: E2E는 이 환경에서 불가; **Node 전용 검증**은 `npm run verify:085`로 가능.
- **전수 조사**: 스텝 중복 3건 확인됨; KPA-085 관련 "대여권 n장 소장권 n장 > 아이콘" 스텝은 단일 정의.
- **추가 조치**: KPA-085에서 Multiple definitions를 피하려면 "사용자가 우측 상단의 프로필 아이콘을 클릭한다" 스텝을 한 곳에서만 정의하도록 정리하는 것이 필요함 (common.auth vs kpa-008-zerostep).
